== What is a 'Disaster' and When Should This Checklist Be Used

*A Disaster*

The basic definition is that the outage _will cause us to break our Service Level Agreement (SLA, *provide link if available*)_ if left alone.

IMPORTANT: Here is where you need a matrix or method for determining the priority of the current error(s) between 1-5, with Priority 1 and 2 being disaster-level. Outage-level errors are those that impact X% of users' ability to interact with Core Functionality of the application.

=== Core Functionality

To determine the criticality of any outage, you need to know if it impacts 'Core Functionality' of the application. Here is the Core Functionality of a Basic Application (*update with the Core Functionality of any given system/application*):

* Access and Authentication
** This includes SSO, if supported
* Accurate Permissions/Authorization (no unauthorized access to data)
* Users can perform CRUD (Create, Read, Update, Delete) for core data
* Users can perform integration actions to external software that we support
* Users with the correct permissions can add/delete users
* API-users can use all documented externally-available endpoints as documented
