== Windows

I‚Äôll need a Unix-like dev environment to develop. I am not a fan of Windows nor am I super familiar with it, so everything takes me much longer to figure out. 

To get Windows to be as much like a Unix-like system as possible, the most appropriate option is probably the Windows Subsystem for Linux (WSL). To set up WSL, I‚Äôll [need to install it](https://ubuntu.com/tutorials/install-ubuntu-on-wsl2-on-windows-10#1-overview), which can be done via a command prompt (either `Powershell` or `cmd`). Generally, you can just use a command like

`wsl --install ubuntu`

This will install the Ubuntu subsystem.

From there, you should be able to see a "Ubuntu" startup item in your Start Menu. You can use this to launch a WSL2 terminal.

From there, you should use `git` to clone one of the team's repositories into the directory of your choosing (this will likely require you to create an SSH key).

From there, you can `cd` into the folder that was just created (you can use `ls` to view the file structure and see the folder you just created) and then use `code .` to open VS Code in WSL2. This will properly link your subsystem's file system with VS Code.

If you successfully linked WSL2, you should see this in the lower-left corner of your VS Code instance.

Linking the WSL2 file system with VS Code properly is very important! If you attempt to open the repository from within Windows itself, simple actions such as `yarn install` will take much longer to complete than normal.
